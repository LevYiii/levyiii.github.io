Github+Hexo搭建个人网站（二）

发表于 2019-02-24 |

前言

在前一章中我们的网站已经可以在网络上搜索到了，但是网址是GitHub提供的固定格式：用户名.github.io 如果以后每次进入个人网站都输入此地址，很显然是很麻烦的。何不尝试将此域名与自己注册的个性域名绑定在一起呢，这样在浏览器中输入自己的个性域名，就能找到个人网站了。因为本人是通过阿里云购买注册域名，因此在本文中都是以阿里云平台为例

绑定域名

登录阿里云->控制台->域名服务

找到要绑定的域名，进入解析


添加如下三条解析：

记录类型：CNAME
主机记录：www
记录值：用户名.github.io

记录类型：A
主机记录：@
记录值：192.30.252.153

记录类型：A
主机记录：@
记录值：192.30.252.154

登录GitHub->进入仓库->设置->在Custom domain栏中输入绑定的域名后保存

进入本地blog/source目录->创建记事本文件并在内容中输入域名->保存为所有文件类型

在blog目录下打开命令行，输入如下命令：

hexo clean

hexo g

hexo d

此时打开浏览器输入域名就能访问到个人网站

(本人的网页做出过修改，不是默认页面)
到这里为止，个人网站可以算是正式上线，如果不喜欢默认主题，可以修改主题–主题传送门

更改主题

主题传送门
以Next主题为例

在blog目录中打开命令行输入：

git clone https://github.com/iissnan/hexo-theme-next themes/next

这条将Next主题下载到blog目录的themes主题下的next文件夹中

打开站点配置文件_config.yml，如下图修改主题为Next

打开主题配置文件_config.yml，找到Scheme Settings,next主题的三个样式，依个人喜好选择样式（去除#号即可）

重新部署网站

hexo g
hexo d

打开浏览器输入域名查看效果

##小结##
到这里位置，个人网站已经搭建完成。除了搭建网站外，更重要的是如何充实网站内容。下一篇将会记录发布文章全过程
